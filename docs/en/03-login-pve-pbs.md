# 🔌 Step 3: Installing the Integration in Home Assistant

To visualize the data (including hardware temperatures), we will use the **Proxmox Extended Sensors** integration.

## 1. Installation via HACS
As this is a custom integration, we must first add it to our HACS store:

1. Go to **HACS > Integrations**.
2. Click on the three dots in the top right corner and select **Custom repositories**.
3. Paste the URL of this repository: `https://github.com/Javisen/proxmox_sensors/`
4. In **Category**, select `Integration` and click **Add**.
5. Search for the integration, install it, and **restart Home Assistant**.

## 2. Configuring the Integration
Once restarted, follow these steps:

1. Go to **Settings > Devices & Services**.
2. Click on **Add Integration** and search for `Proxmox Extended Sensors`.

## 3. Connection Data
The login form is very straightforward, but pay attention to these details:

* **Host:**
    * If on your local network: Enter only the IP (e.g., `192.168.1.50`). **No need to include the port**.
    * If accessing from outside: Enter your domain. **Do not write `http://` or `https://`**, the integration detects this automatically.
* **Server Type:** Select between **PVE** (Proxmox Virtual Environment) or **PBS** (Proxmox Backup Server).
* **Use Token:** Select whether you will use the API Token created in Step 2 or a traditional login.

### Option A: Login with User (No Token)
If you prefer not to use a token, fill in these fields:
* **User:** Always use the format `user@realm` (example: `homeassistant@pve` or `root@pam`).
* **Password:** The user's password.
* **Node Name:** The name of your Proxmox node (as it appears in the tree on the left in the Proxmox web interface).

### Option B: Login with Token (Mandatory for PBS)
If you prefer to use a token, fill in these fields:
* **User:** Always use the format `user@realm` (example: `homeassistant@pve` or `root@pam`).
* **token_id:** The identifier name you gave the token (example: `ha-token`). Do not confuse this with the full ID.
* **Token_secret:** The character string (secret) generated by Proxmox.

## ✅ Entity Selection (PVE ENVIRONMENT ONLY)
Once you click submit, the integration (in PVE mode) will scan your server and allow you to choose what you want to monitor:
* **VMs:** Specific virtual machines.
* **CTs:** LXC containers.
* **Physical Disks:** Connected hard drives and SSDs.
* **Storages:** Storage partitions and their free space.

> [!TIP]
> **Only select what you truly need.** This will keep your Home Assistant clean and ensure better performance.

## ⚠️ Important note for PBS in shared environments (e.g., Tuxis)

If you use the free version of **Tuxis** or similar managed PBS providers, you must understand that the integration will have significant limitations. This is because your PBS instance runs in a **shared (Multi-tenant) environment** and not on a dedicated server.

### Why won't you see all sensors?
Unlike a local Proxmox setup, in these services:
* **No Real Hardware Access:** You do not have access to the actual filesystem or direct physical storage.
* **Hidden Infrastructure:** You cannot monitor the backend (Ceph/ZFS) they use, as it is owned by the provider.
* **Privacy and Security:** The provider blocks access to global system metrics to prevent one client from inferring information about other users or the total load of their infrastructure.
* **No Root Permissions:** Since you don't have access to the system root (`/`), data such as node temperature sensors or fan speeds cannot be extracted.

**Result:** In these cases, the integration will not show anything or only sensors without information. We will work to try to show personal datacenter data in future versions.
