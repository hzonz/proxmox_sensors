# 🔌 Шаг 3: Установка интеграции в Home Assistant

Для визуализации всех данных (температуры, аппаратные датчики, диски, PBS, ВМ и КТ) мы будем использовать интеграцию **Proxmox Extended Sensors**.

---

## 1. Установка через HACS

Поскольку это пользовательская интеграция, сначала необходимо добавить её в HACS:

1. Перейдите в **HACS → Интеграции**
2. Нажмите на **три точки** (вверху справа)
3. Выберите **Пользовательские репозитории**
4. Добавьте этот репозиторий:
   `https://github.com/Javisen/proxmox_sensors/`
5. В разделе **Категория** выберите `Интеграция`
6. Установите интеграцию и **перезапустите Home Assistant**

---

## 2. Добавление интеграции

После перезапуска:

1. Перейдите в **Настройки → Устройства и Сервисы**
2. Нажмите **Добавить интеграцию**
3. Найдите **Proxmox Extended Sensors**

---

## 3. Настройка подключения

### 🔹 Хост
- **Локальная сеть:** `192.168.1.50`
- **Внешний доступ:** `proxmox.мойдомен.com`

> Нет необходимости указывать `http://` или `https://`. Это определяется автоматически.

---

### 🔹 Тип сервера
- **CLUSTER** → Кластер Proxmox
- **PVE** → Proxmox Virtual Environment
- **PBS** → Proxmox Backup Server

---

### 🔹 Метод аутентификации

- **Пользователь + пароль** → только на PVE и кластере
- **API токен** → Обязателен на PBS

---

## 🔐 Вариант А: Пользователь и пароль (только PVE)

Поля:

- **Пользователь:** `пользователь@realm`
  - Пример: `homeassistant@pve`
- **Пароль:** пароль пользователя

> 💡 Начиная с V3, узел определяется автоматически. Вводить его вручную не требуется.

---

## 🔐 Вариант Б: API токен (рекомендуется)

Поля:

- **Пользователь:** `пользователь@realm`
- **ID токена:** только имя → `ha-token`
- **Секрет токена:** секрет, сгенерированный в Proxmox

> ⚠️ Не используйте формат `пользователь@pve!токен`

---

## 🧠 Выбор ресурсов (PVE)

После подключения интеграция автоматически обнаружит доступные ресурсы.

Вы сможете выбрать:

- Виртуальные машины (ВМ)
- Контейнеры (КТ)
- Физические диски
- Хранилища

> 💡 Выбирайте только необходимое, чтобы поддерживать Home Assistant в чистоте и эффективности.

## ⚠️ Примечание о PBS в управляемых средах

Если вы используете **управляемый или мультитенантный** PBS (Tuxis, Hetzner и т.д.):

- У вас не будет доступа к аппаратным датчикам
- Вы не увидите температуры или физические диски
- Не будет метрик узла

Это нормально, потому что:

- У вас нет доступа к реальному оборудованию
- Провайдер ограничивает систему
- Не существуют низкоуровневые разрешения

**Результат:**
Будут отображаться только ограниченные данные хранилища (datastore).
