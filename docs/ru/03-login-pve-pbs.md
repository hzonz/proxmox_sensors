# 🔌 Шаг 3: Установка интеграции в Home Assistant

Для просмотра данных (включая температуры, аппаратные датчики, диски, PBS, ВМ и контейнеры) мы будем использовать интеграцию **Proxmox Extended Sensors**.

[Визуальное руководство по установке](#-Визуальное-руководство-по-установке)

---

## 1. Установка через HACS

Поскольку это пользовательская интеграция, сначала мы должны добавить её в HACS:

1. Перейдите в **HACS → Интеграции**  
2. Нажмите на **три точки** (вверху справа)  
3. Выберите **Пользовательские репозитории**  
4. Добавьте этот репозиторий: `https://github.com/Javisen/proxmox_sensors/`
5. В **Категории** выберите `Интеграция`  
6. Установите её и **перезапустите Home Assistant**

---

## 2. Настройка интеграции

После перезапуска:

1. Перейдите в **Настройки → Устройства и службы**  
2. Нажмите **Добавить интеграцию**  
3. Найдите **Proxmox Extended Sensors**

---

## 3. Данные подключения

Форма простая, но есть важные детали:

### 🔹 Хост
- **В локальной сети:** только IP-адрес → `192.168.1.50`  
*(Не добавляйте порт или http/https)*  
- **Извне:** ваш домен → `proxmox.moydomen.com`  
*(Интеграция автоматически определяет http/https)*

### 🔹 Тип сервера
- **PVE** → Proxmox Virtual Environment  
- **PBS** → Proxmox Backup Server  

### 🔹 Метод аутентификации
- **Традиционный логин** (только PVE)  
- **Токен API** (обязательно для PBS)

---

## 🔐 Вариант А: Вход с пользователем (без токена)

Действительно только для **PVE**.

Поля:

- **Пользователь:** `пользователь@realm`  
Примеры:  
- `homeassistant@pve`  
- `root@pam`  
- **Пароль:** пароль пользователя  
- **Имя узла:** имя узла (как оно отображается в Proxmox)

---

## 🔐 Вариант Б: Вход с токеном (рекомендуется и обязательно для PBS)

Поля:

- **Пользователь:** `пользователь@realm`  
- **token_id:** только имя токена → `ha-token`  
*(Не пишите `пользователь@pve!token`)*  
- **Token_secret:** Secret, сгенерированный Proxmox  

---

## ✅ Выбор сущностей (только в PVE)

После подключения интеграция просканирует ваш сервер, и вы сможете выбрать, что отслеживать:

- **Виртуальные машины**  
- **Контейнеры**  
- **Физические диски**  
- **Хранилища**

> [!TIP]  
> Выбирайте только то, что вам нужно, чтобы Home Assistant оставался чистым и быстрым.

## ⚠️ Важное примечание для PBS в общих средах (Tuxis, Hetzner и т.д.)

Если вы используете **управляемый** или **мультитенантный** PBS, например Tuxis Free PBS:

- Вы не увидите аппаратные датчики  
- Вы не увидите температуры  
- Вы не увидите физические диски  
- Вы не увидите метрики узла  

Это нормально, потому что:

- У вас нет доступа к реальному оборудованию  
- Провайдер скрывает инфраструктуру  
- У вас нет прав root  
- Вы не можете получить доступ к реальной файловой системе  

**Результат:**  
Интеграция будет показывать только пустые датчики или отсутствие данных.  
В будущих версиях мы постараемся отображать пользовательские метрики хранилища данных.
