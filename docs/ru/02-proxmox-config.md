# 🔐 Шаг 2: Настройка пользователей и прав доступа

**Чтобы Home Assistant мог безопасно взаимодействовать с Proxmox, рекомендуется не использовать пользователя root. Мы создадим специального пользователя и назначим ему необходимые права доступа для 100% работоспособности интеграции.**

> ⚠️ **ВАЖНО:**  
> Из-за расширенных функций интеграции (управление виртуальными машинами/контейнерами, индивидуальные и массовые резервные копии, действия с PBS…) необходимо назначить **права администратора** как в PVE, так и в PBS.

---

## 1. Разница между PVE и PBS

### **Proxmox VE (PVE)**
- Вы можете использовать **Пользователя/Пароль** или **Токен API**.  
- Пользователь должен иметь роль **PVEAdmin**.

### **Proxmox Backup Server (PBS)**
- **Обязательно** использовать **Токен API**.  
- Пользователь должен иметь роль **Administrator** (в PBS нет подходящей промежуточной роли).

---

## 2. Создание пользователя

1. Перейдите в **Datacenter → Permissions → Users**  
2. Нажмите **Add**  
3. Настройте:  
   - **User:** `homeassistant`  
   - **Realm:** `pve`  
   - **Password:** только если вы будете использовать вход по паролю в PVE  
4. Сохраните изменения

---

## 3. Назначение правильной роли

1. Перейдите в **Datacenter → Permissions**  
2. Нажмите **Add → User Permission**  
3. Настройте следующие поля:

### ✔ Для PVE:
- **Path:** `/`  
- **User:** `homeassistant@pve`  
- **Role:** `PVEAdmin`  

### ✔ Для PBS:
- **Path:** `/`  
- **User:** `homeassistant@pve`  
- **Role:** `Administrator`  

> 💡 **Почему `/` необходимо:**  
> Интеграции требуется глобальный доступ для чтения узлов, виртуальных машин, контейнеров, дисков, хранилищ данных и задач.

---

## 4. Генерация токена API (Обязательно для PBS)

1. Перейдите в **Datacenter → Permissions → API Tokens**  
2. Нажмите **Add**  
3. Настройте:  
   - **User:** `homeassistant@pve`  
   - **Token ID:** `ha-token`  
   - **Privilege Separation:** **снят флажок**  
   - **Expire:** **Never**  
4. При создании токена Proxmox отобразит:  
   - **Token ID**  
   - **Secret** (показывается только один раз)

> [!WARNING]
> **Скопируйте "Secret" сейчас и сохраните его в безопасном месте.** Как только вы закроете это окно, Proxmox больше никогда не покажет его по соображениям безопасности.

> [!TIP]
> ### 💡 Забыли скопировать Secret?
> Не волнуйтесь. Хотя Proxmox больше не покажет его по соображениям безопасности, вам не нужно удалять токен и начинать заново:
> 
> 1. В списке **API Tokens** выберите токен, который вы уже создали.
> 2. Нажмите кнопку **Regenerate**.
> 3. Система немедленно аннулирует старый ключ и предоставит вам **новый Secret**.
> 
> *Помните, что если вы восстановите Secret, вам нужно будет обновить его в конфигурации Home Assistant, чтобы интеграция могла восстановить подключение.*
