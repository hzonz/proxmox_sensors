# 🔐 Шаг 2: Настройка пользователей и прав доступа
**Для безопасного взаимодействия Home Assistant с Proxmox рекомендуется не использовать пользователя root. В этом руководстве мы создадим учетную запись с правами «только для чтения».**

## 1. Разница между PVE и PBS
Перед началом примите во внимание следующее:

* **Proxmox VE (PVE): Вы можете использовать обычный логин/пароль или API Token.**

* **Proxmox Backup Server (PBS): Использование API Token обязательно. Традиционные методы входа часто не срабатывают из-за ограничений безопасности или настроек прав доступа к Datastore.**

---

## 2. Создание роли (Права доступа)
**«Роль» определяет, что именно может делать интеграция.**

1. Перейдите в Datacenter > Permissions > Roles.

2. Нажмите Create и введите имя HA-Monitor.

---

## 3. Настройка привилегий роли
Роль определяет возможности интеграции.
1. Перейдите в **Datacenter > Permissions > Roles**.
2. Нажмите **Create** и назовите её `HA-Monitor`.
3. Выберите следующие привилегии (**Privileges**):
    * `Sys.Audit`: Позволяет видеть состояние узла (CPU, RAM).
    * `VM.Audit`: Позволяет видеть состояние виртуальных машин и контейнеров.
    * `Datastore.Audit`: Позволяет видеть дисковое пространство.

---

## 4. Создание пользователя
1. **Перейдите в Datacenter > Permissions > Users.**

2. **Нажмите Add.**

3. **User:** `homeassistant` (вы можете оставить realm как pve).

4. **Установите надежный пароль, если планируете использовать этот метод для PVE.**

---

## 5. Назначение роли
**Необходимо указать Proxmox, что созданный пользователь обладает созданной нами ролью:**

1. Перейдите в **Datacenter > Permissions**.
2. Нажмите **Add > User Permission**.
3. Настройте следующие поля:
    * **Path:** `/` (Это очень важно, чтобы интеграция могла видеть весь сервер).
    * **User:** `homeassistant@pve` (или созданный вами пользователь).
    * **Role:** `HA-Monitor`.

---

## 6. Создание API Token (Обязательно для PBS)
**Если вы собираетесь мониторить PBS или предпочитаете не использовать пароли в PVE, выполните следующие шаги:**

1. Перейдите в **Datacenter > Permissions > API Tokens**.
2. Нажмите **Add** и заполните форму:
    * **User:** Выберите пользователя `homeassistant`.
    * **Token ID:** `ha-token` (можно выбрать любое имя).
    * **Privilege Separation:** ⚠️ **СНИМИТЕ эту галочку**. Если оставить её включенной, токен не унаследует права пользователя, и интеграция не будет работать.
3. После нажатия на **Add** откроется окно с двумя важными параметрами:
    * **Token ID:** (Например: `homeassistant@pve!ha-token`).
    * **Secret:** (Длинная строка из букв и цифр).

> [!WARNING]
> **Скопируйте «Secret» прямо сейчас и сохраните его в надежном месте.** Как только вы закроете это окно, Proxmox больше никогда его не покажет в целях безопасности.


> [!TIP]
> ### 💡 Забыли скопировать Secret?
> Не волнуйтесь. Хотя Proxmox не показывает его повторно, вам не нужно удалять токен и начинать всё сначала:
> 
> 1. В списке **API Tokens** выберите уже созданный токен.
> 2. Нажмите кнопку **Regenerate**.
> 3. Система немедленно аннулирует старый ключ и выдаст вам **новый Secret**.
> 
> *Помните, что после регенерации Secret вам нужно будет обновить его в настройках Home Assistant, чтобы восстановить соединение.*
