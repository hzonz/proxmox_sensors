# 🚀 Proxmox Extended Sensors

## Введение

**Proxmox Extended Sensors — это интеграция для Home Assistant, предназначенная для обеспечения расширенного мониторинга и полного управления Proxmox VE и Proxmox Backup Server (PBS).**

В отличие от решений, основанных исключительно на метриках, эта интеграция внедряет подход, ориентированный на **полезную информацию (инсайт)** , позволяя понять не только то, что происходит в системе, но и то, как она на самом деле работает.

Она обеспечивает полную видимость инфраструктуры и добавляет возможности прямого управления узлами, виртуальными машинами, контейнерами, хранилищами и службами резервного копирования.

---

## 🧠 Системная аналитика (System Insight) (V3/V4)

Начиная с версии 3, интеграция эволюционировала от набора технических метрик к системе наблюдаемости, ориентированной на инфраструктуру.

V4 представляет датчики, способные интерпретировать глобальное состояние узла и преобразовывать сложные метрики в полезную и действенную информацию:

- **Узел Proxmox** → глобальное состояние узла (`Excellent`, `Warning`, `Critical` и т.д.) с обогащёнными атрибутами инфраструктуры
- **Оценка узла** → числовая оценка производительности и общего состояния системы
- **Средняя нагрузка (1м / 5м / 15м)** → фактическая нагрузка на хост
- **Ожидание ввода-вывода** → обнаружение давления и насыщения диска
- **Использование ЦП на ядро** → доступно для узлов, ВМ и контейнеров
- **Сетевая телеметрия узла** → интеллектуальный расчёт агрегированного трафика RX/TX от ВМ и КТ
- **Расширенная информация о хранилище** → состояние, ёмкость и детальные метрики физических дисков и хранилищ

Эти датчики позволяют обнаруживать узкие места, выявлять деградацию системы и создавать гораздо более интеллектуальные автоматизации без необходимости в дополнительных внешних инструментах.

- Глобальный мониторинг кластера Proxmox
- Расширенное обнаружение смонтированных дисков (CIFS/NFS/local)
- Интеллектуальная телеметрия сети и хранилищ

- Узлов
- Виртуальных машин (QEMU)
- Контейнеров (LXC)

- Управляющие действия из Home Assistant
- Встроенные службы резервного копирования
- Полная совместимость с PBS (включая дедупликацию)
- Безопасная аутентификация с помощью токенов
- Чистая и последовательная структура сущностей
- Оптимизированные обновления и низкое потребление ресурсов

## 💾 Службы резервного копирования (ВМ и КТ)

Интеграция позволяет создавать резервные копии непосредственно из Home Assistant, полностью совместимые с Proxmox VE и PBS.

### 🟦 Индивидуальное резервное копирование

- Поддержка нескольких ID (через запятую)
- Режимы: snapshot / suspend / stop
- Сжатие: zstd / gzip / lzo / none
- Совместимо с PBS и дедупликацией

### 🟩 Массовое резервное копирование

- Резервное копирование всех ресурсов узла
- Контроль параллельности и времени выполнения
- Идеально для автоматизации
- Совместимо с крупной инфраструктурой

Резервные копии автоматически именуются следующим образом:

```text
HA-{{vmid}}-{{guestname}}
```

Полностью совместимы с PBS, включая дедупликацию и существующие цепочки.

## 🗄️ Proxmox Backup Server (PBS)

Расширенный мониторинг хранилищ данных (datastore) и задач:

- Общее, свободное использование и процент
- Коэффициент дедупликации
- Статус последней резервной копии
- Ошибки и сводка задач
- Состояние сборщика мусора (Garbage Collector)
- Детальная информация о задачах

---

## 🎛️ Управляющие действия (PVE & PBS)

**Узел:**
- Выключение / Перезагрузка / Wake-on-LAN

**Виртуальные машины:**
- Запуск / Остановка / Выключение / Перезагрузка / Сброс
- Пауза / Возобновление / Гиberнация

**Контейнеры:**
- Запуск / Остановка / Выключение / Перезагрузка

**PBS:**
- Сборщик мусора (Garbage Collector)
- Очистка (Prune)
- Проверка (Verify)
- Синхронизация (Sync)

---

## 🎨 Организация и структура

- Датчики автоматически группируются в:
  1. Кластер
  2. Узел
  3. Физические диски
  4. Виртуальные машины
  5. Контейнеры
  6. Хранилища / Datastore
  7. PBS и задачи

- Последовательные и чёткие имена для упрощения панелей мониторинга и автоматизаций

## 🧩 Установка

### 🔹 Через HACS (рекомендуется)

1. Откройте **HACS → Интеграции**
2. Добавьте пользовательский репозиторий
3. Найдите **Proxmox Extended Sensors**
4. Установите и перезапустите Home Assistant
5. Добавьте интеграцию из настроек

### 🔹 Ручная установка

1. Скопируйте в `/config/custom_components/proxmox_sensors`
2. Перезапустите Home Assistant
3. Добавьте интеграцию
